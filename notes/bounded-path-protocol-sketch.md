# Bounded-Path Pilot Protocol Sketch

## Contact

Brett Reynolds  
Humber Polytechnic & University of Toronto  
brett.reynolds@humber.ca

## Objective

Test whether dogs select defecation sites relative to path edges and junctions in systematic ways, and whether those choices shift when off-path access is constrained by bounded paths.

## Design

- Within-dog repeated-measures pilot
- 15-20 adult dogs
- 2-4 sessions per dog
- One first-defecation event scored per session
- Standardized walk route through path conditions that differ only in boundedness

## Path conditions

- Open path: dog can step off freely
- Low-bounded path: stepping off is easy but mildly costly or effortful
- High-bounded path: stepping off is difficult or functionally blocked

All conditions should keep path width, length, and general substrate as constant as possible.

## Site options

The test area should include:
- straight segments
- at least one junction
- accessible off-path grass or substrate in the open and low-bounded conditions
- clear physical mapping of path centerline and edge

The study should be run at an existing collaborator site if possible: fenced field, outdoor test lane, shelter exercise area, or other controlled outdoor dog-testing environment.

## Subjects

- Adult domestic dogs
- Exclude dogs with gastrointestinal illness or mobility issues
- Prefer dogs that reliably defecate during a routine walk
- Record sex, age, neuter status, breed or type, and usual walk routine

## Handling protocol

- Same handler protocol across dogs and sessions
- Fixed leash length
- Standardized pace
- Minimal verbal input
- No deliberate steering toward path edge or off-path area

If multiple handlers are necessary, handler identity should be recorded and balanced if possible.

## Session structure

1. Pre-session check:
   - confirm the dog is fit to walk
   - record time since last meal and last defecation if known
   - record weather and surface conditions
2. Standardized walk through the test path
3. Record only the first defecation event
4. End the scored portion of the session after the first event

## Primary outcomes

- lateral distance from path centerline
- categorical location: path center, path edge, off-path
- distance to nearest junction
- whether the dog attempted to step off the path before defecating

## Secondary outcomes

- latency to defecation
- sniffing rate or duration during the final search phase
- circling before defecation
- preceding urination
- leash tension or obvious handler influence
- surface and weather conditions

## Video and mapping

- Use fixed cameras rather than drones
- Minimum of 2 synchronized cameras covering the path and adjacent ground
- Map the path network before testing
- Mark path edges and junctions clearly for later coding
- Record event coordinates from video or post-event site mapping

## Coding window

Define the final 60-120 seconds before defecation as the search phase and code:
- sniffing
- circling
- pace reduction
- pauses
- attempts to leave the path
- urination before defecation

## Main interpretive rule

On-path defecation should not automatically be treated as path preference. It should be interpreted relative to accessible alternatives. If off-path areas are blocked or effectively unusable, the event should be coded as constrained by accessibility.

## Analysis plan

- Treat each defecation event as a site-selection problem
- Compare chosen sites with available sites visited during the search phase
- Use a mixed-effects or conditional logistic framework
- Include dog ID as a random effect
- Include boundedness condition, junction proximity, and accessibility as core predictors

## Immediate feasibility questions for collaborators

- Is the question publishable at pilot scale?
- Which subject population is best for a first study: owned dogs, shelter dogs, or free-ranging dogs?
- Can an existing facility support the bounded-path manipulation cleanly?
- What level of behavioral coding is realistic without overbuilding the study?
